Team Quiz — How to run it

A live quiz for onboarding sessions. Everyone joins on their phone or laptop, gets split into two random teams, and answers questions together — with the correct answer and a short explanation revealed after every round.

FOR THE HOST

Running a session

  1. Open the host panel at /host.html and sign in with your admin username and password. A 4-digit game code appears at the top — this is your room.
    No account yet? The super-admin creates one for you at /admin.html.
    Keep this tab to yourself — it shows the correct answers. Several hosts can run their own games at once; each gets its own code.
  2. Share the game code (or the copy-paste link) with your players. They enter it on the join screen to land in your game, not someone else's.
    Use “Close game” when you're done to free the room.
  3. Pick a question set from the dropdown (e.g. Java & DevOps, AWS, Git). The counter shows how many questions in that set are still unused.
    Since v1.1.0 you can switch set at any time — mid-game it applies from the next question.
  4. Share the main site link with the room and wait for players to appear in the lobby.
  5. Press Draw teams to split everyone randomly into two teams. Use Redraw if you want a reshuffle.
  6. Press Start game. The first question appears — read it aloud for the room.
  7. When both teams have voted the answer reveals automatically. If a team is slow, you can force it with Reveal. Read out the explanation.
  8. Press Next question to continue. First team to 3 points (and ahead) wins.
  9. New match returns to the lobby keeping the players; Reset question history lets already-used questions in the current set appear again.
FOR PLAYERS

Playing

  1. Open the site link the host shares and enter your name.
  2. Wait in the lobby — the host will draw everyone into two teams.
  3. Read the question and options on your screen and talk it over with your team.
  4. One person taps your team's answer. There's one vote per team, and it locks once cast — so agree first.
  5. Once both teams have voted, the correct answer and an explanation appear. Learn, then move on.
  6. A correct answer scores a point. First team to 3 points, and ahead, wins.
Good to know
  • It's team-based: the discussion is the point, not just the score.
  • A tie at the winning score keeps playing until one team pulls ahead.
  • Each set remembers what it has already asked, so questions don't repeat until it runs out.
  • Joining late is fine — you'll be placed on a team at the next draw.
  • Each question has a timer (default 60s, host-configurable, 0 = off). When it runs out, the answer is revealed — a team that hasn't voted scores nothing.
  • Questions have a difficulty tier (medium / hard / pro — shown next to the topic). The host can filter tiers at any time, e.g. medium+hard for newer groups.
  • The host password can be changed from the host panel itself (current + new, stored server-side as a salted hash).
  • The host can end the match at any time: the leading team wins, or a tie is declared. Either way you get the full recap.
  • When a match ends, everyone sees a recap: every question, the correct answer in green, wrong picks in red, and the explanation.
